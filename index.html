UAS PAM Pengembangan Aplikasi Mobile (Kotlin)

Buat aplikasi dengan persyaratan dan pertimbangan berikut ini: Tech Stack API Fungsi Dokumentasi Bonus Pengembangan

  • Tech Stack *25%

    1. Pola Arsitektur menggunakan MVVM (Model - View - View Model)
    2. SDK Target = 34
    3. Basis Data Lokal dengan Room DB
    4. Layanan API dengan Retrofit
    5. Tambahkan Interceptor HTTP untuk Pencatatan API menggunakan Chucker
    6. Navigasi dengan Navigation
    7. Paginasi menggunakan Paging2 / Paging3
    8. Pengujian UI aliran dengan API menggunakan MockWebServer
    9. Pengujian Unit beberapa metode / fungsi menggunakan Mockito dengan Fake, Mock, dan Dummy

  • Implementasi API *15%

    1. Gunakan API publik sebagai sumber data
      Daftar API Publik :
      API Pokemon
      API Digimon
      API TheMovieDB
      API OpenWeatherMap
      API Pengguna GitHub
      API Negara
    2. * API lain selain daftar di atas juga memungkinkan, harap minta melalui discord
    3. * Minimum tiga titik akhir API diimplementasikan

  • Fungsi *40%

    1. Penggunaan OnBoarding dengan Motion Layout
    2. Tampilkan daftar melalui RecyclerView dengan Paginasi (Paging), juga tambahkan fungsi pencarian
    3. Beberapa fungsi dengan Tampilkan daftar, pencarian, dan dapatkan beberapa detail menggunakan API
    4. Tambahkan dan gunakan beberapa fungsi CRUD (Create, Update, Delete) dengan Room DB, misalnya membuat daftar Favorit
    5. Daftarkan dan masuk pengguna dengan otentikasi firebase
    6. Unggah foto dari kamera atau galeri
    7. Implementasikan Crashlytics
    8. Implementasikan Pengujian UI dengan minimal Dua Tes
    9. Implementasikan Pengujian Unit dengan minimal Tiga Metode / Fungsi
    10. Implementasikan Interceptor API Chucker

  • Dokumentasi *20%

    1. Semua Dokumentasi dibuat dalam Bahasa Inggris
    2. Buat Mockup dan Prototyping dari Figma, Template Contoh
    3. Buat Dokumentasi Proposal dalam Format PDF, Template Proposal
    4. Buat Readme.md Informatif di GitHub
      Contohnya : Tentang aplikasi Tech Stack Daftar fungsi Prototipe Figma Berkas GIF Video YouTube tentang aplikasi
    5. Buat Perubahan yang Diunggah Informatif di GitHub
    6. Buat video YouTube yang menggambarkan tentang konten aplikasi

  • Bonus Pengembangan *30%

    1. Kode Bersih (Struktur Proyek, Konvensi Penamaan, Kode Boilerplate Minimum, Pengelolaan Aset)
    2. Pengalaman Pengguna (Animasi, Loading, Aliran)

  • Waktu Kerja

    1. Bentuk tim (3-5 per tim) dan cari ide pengembangan: Minggu Libur: Kamis, 11 April 2024
    2. Presentasi rencana proposal proyek: Minggu ke-2: Kamis, 18 April 2024
    3. Presentasi prototipe (Figma): Minggu ke-3: Kamis, 25 April 2024
    4. Presentasi perkembangan pengembangan: Minggu ke-4: Kamis, 02 Mei 2024
    5. Presentasi aplikasi proyek: Minggu ke-5: Kamis, 09 Mei 2024
    6. Meningkatkan dan Membersihkan Kode, menyelesaikan Readme.md: Minggu ke-6: Kamis, 16 Mei 2024
    7. Presentasi dan membangun File APK: Minggu ke-7: Kamis, 23 Mei 2024
    8. Formulir Pengiriman UAS: Klik di Sini
